Honey 배포 반영 가이드
======================

현재 배포 방식은 PyInstaller onedir 결과물(client/dist/Honey/)을 ZIP으로 묶어
server/releases/에서 제공하는 방식이다. 별도 설치 프로그램은 사용하지 않는다.

주요 파일
---------

- 빌드 스크립트: client/build_zip.bat
- 릴리스 스크립트: client/release/release_honey.ps1
- 빌드 산출물: client/release_dist/Honey-<version>.zip
- 서버 배포 위치: server/releases/Honey-<version>.zip
- 매니페스트: server/releases/version.json
- 서버 라우트: server/honey_routes.py (/honey/version, /honey/download)

배포 명령
---------

    cd F:\COINAPI\report_server\client\release
    .\release_honey.ps1 -Version 3.0.1 -Notes "변경 사항 요약"

또는 client/build_zip.bat을 실행한다.

필수 준비물
-----------

- Python
- client/requirements.txt 패키지
- PyInstaller

설치 예:

    cd F:\COINAPI\report_server\client
    python -m pip install -r requirements.txt
    python -m pip install pyinstaller

동작 요약
---------

1. Honey 시작 시 /honey/version 조회
2. 새 버전이면 /honey/download에서 Honey-<version>.zip 다운로드
3. sha256 검증
4. ZIP 압축 해제
5. 외부 bat가 실행 중인 Honey 종료를 기다린 뒤 앱 폴더에 복사
6. Honey.exe 자동 재실행

주의 사항
---------

- 실행 중인 Honey.exe를 직접 덮어쓰지 않는다.
- version.json은 release_honey.ps1이 자동으로 BOM 없는 UTF-8로 저장한다.
- server/releases/*.zip은 git에 올리지 않는다. version.json만 추적한다.
