# 변환규칙 
## A . 진입점 골격

'write()' 시작에서 'xw.App' 컨텍스트 1개를 열고 끝까지 그안에서 처리. openpyxl 워크북 생성 저장 호출 모두 제거 . distribution phase 2 의 '_write_distribuition_xlwgins' 가 별도로 앱을 다시 여는 부분도 제거하고 같은 'wb' 객체를 '_write_distribution()' 에 그대로 전달

```python
import xlwings as xw
with xw.App(visible = False, add_book=False) as app:
	app.Calculation = 'manual'

	wb.save(out_path)

```

C. Table 시트
각 '_Fill_"함수 시그니처('ws,result') 는 유지하고 본문만 xlwings 로 치환. 
이미 분석은 끝났으니 셀 기입 + 스타일만 바뀐다.

openpyxl -> xlwings 매핑 표

'wb = openpyxl.Workbook()' -> wb = app.books.add()
wb.create_sheet(name) -> wb.sheets.add(name, after=wb.sheets[-1])
wb.remove(ws) / del wb[name] -> wb.sheets[name].delete()
ws.cell(row=r, column=c, value=v) -> ws.range((r,c)).value = v
일괄 기입(2D 리스트) -> ws.range((r.c)).value = rows
ws.merge_cells(B3:D3:) -> ws.range(B3:D3).merge()
ws.column_dimensions['B'].width = w -> ws.range(B:B).column_width = w
ws.row_dimensions[r].height = h -> ws.range(f'{r}:{r}).row_height = h
ws.sheet_view.zoomScale = 80 -> ws.api.Parent.windows(1).Zoom = 80 (또는 wb.app.api.Activewindow.zoom)
wsheet_view.showgridlines = False -> ws.api.Parent.Windows(1).DisplayGridlines = False

스타일은 EXcel Com ('.api')로 : 
기존 모듈 상단 색상/폰트 상수는 유지하되 적용함수만 xlwings 로. ARGB 8자리 뒤 6자리만 빼서 _rgb_int 입력

** 범위 단위 일괄 적용 ** 으로 셀 단위 호출 최소화 예 : 헤더 1행 전체에 '_apply_hdr_style_xw(ws.range((3,2),(3,2+ncols-1)))'

Distribution 차트는 그대로
_write_distribution_xlwings() 의 "App/Workbook 직접 열기" 부분은 제거
호출 시그니처를 (wb, result, colors, ...) 로 바꿔 상위 컨텍스트의 wb 를 그대로 받게 함.
zoom 이나 grid line 는 ㅅ단일 세션안에서

diff 시트
diff_cpk_sepc : _fill_cpk_rows(ws,rows) 본문도 c 규칙대로 xlwgins 변환
diff_dst_spec : distribution 챁 ㅡ그리는 함수에 위임

정리
from openpyxl ... import 전부 제거 copy, path 등은 남김
wb.save() 는 컨텍스트 안에서 1회
_finalize_openpyxl_sheet_layouts, normalize reportsheetnames 의 openpyxl 의존 부분도 xlwigns 옮기거나 호출 시점만 단일 세션내로 이동.
