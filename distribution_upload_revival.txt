============================================================
Distribution(산포) 서버 업로드 재활성화 — 작업 지시 (외부 담당자용)
============================================================

이 문서는 외부 담당자에게 그대로 전달하는 작업 지시(프롬프트)다.


------------------------------------------------------------
[Context] 왜 하는가
------------------------------------------------------------
Honey 클라이언트가 만든 리포트의 distribution(산포) 시트를 PNG 한 장으로
합성해 서버에 업로드 → 웹 세션 상세에 "Distribution" 탭으로 표시하는 흐름이
과거에 있었으나 현재 4지점에서 의도적으로 끊겨 있다.
변환 함수(PyMuPDF) / 서버 저장소 / 서빙 라우트는 이미 살아있고,
클라<->서버 "배선"과 뷰 탭만 다시 연결하면 된다.

추가로 distribution PNG가 너무 크고 가로로 짤려서 올라가는 문제가 있다.
원인은 PDF export 시 시트의 인쇄영역/페이지설정을 그대로 따르기 때문이며,
export 전에 page setup(한 페이지 폭 맞춤)을 지정하고 렌더 DPI를 상수화해
해결한다.

불변 규칙(CLAUDE.md):
- 서버는 openpyxl/Excel 미사용. distribution PNG는 클라이언트가
  Excel COM + PyMuPDF로 만들어 전송한다.
- analysis_key 산출식 / DB 스키마는 건드리지 않는다.
- report_generator/ (분석/차트 생성)는 수정하지 않는다.

== D. 짤림/크기 조정 - client/transport/chart_export.py
       export_distribution_png() ==

채택 방식: 한 페이지 폭에 맞춤 + DPI 상수화.

- 모듈 상단에 상수 추가 (담당자/사용자가 여기만 만지면 조정 가능):

    _DIST_DPI_SCALE = 1.5   # PDF->PNG 렌더 배율. 키우면 선명/큼, 줄이면(예 1.0) 작아짐.

- ws.ExportAsFixedFormat(...) 호출(417줄) 직전에 page setup 지정:

    try:
        ps = ws.PageSetup
        ps.Orientation = 2        # xlLandscape (가로)
        ps.Zoom = False           # Zoom 끄고 FitTo 사용
        ps.FitToPagesWide = 1     # 가로 1페이지로 -> 폭 짤림 제거
        ps.FitToPagesTall = False # 세로는 여러 페이지 허용(기존 수직 합성이 처리)
    except Exception:
        pass                      # page setup 실패해도 export 는 진행

- 렌더 배율 하드코딩(443/445/451줄의 fitz.Matrix(1.5, 1.5))을
    fitz.Matrix(_DIST_DPI_SCALE, _DIST_DPI_SCALE)
  로 교체.

튜닝 가이드 (사용자/담당자용):
- "너무 큼"      -> _DIST_DPI_SCALE 를 낮춤 (1.5 -> 1.0 또는 0.8).
- "가로로 짤림"  -> FitToPagesWide = 1 이 핵심. 그래도 글씨가 너무 작으면
                    FitToPagesWide 를 2로 늘리거나 여백(ps.LeftMargin /
                    RightMargin 등)을 줄임.
- "세로로 너무 길게 합성됨" -> FitToPagesTall = 1 로 두면 한 장에 다
                    욱여넣지만 글씨가 작아짐 (trade-off).

[검증 - E2E]
------------------------------------------------------------
1. 클라이언트: PyMuPDF(pip install PyMuPDF) + Excel 설치된 PC 에서
   distribution 시트가 있는 xlsx 를 업로드.
2. 서버 응답 JSON 의 distribution_combined: true 확인.
   (S3 미설정 시 uploads/.../dist_combined/<analysis_key>.png 생성 확인)
3. 브라우저 .../pe/report/view/<session_id> -> Distribution 탭 클릭 ->
   합성 PNG 가 폭 안 짤리고 적당한 크기로 표시되는지 확인.
4. _DIST_DPI_SCALE / FitToPagesWide 값을 바꿔가며 크기/짤림 재확인.
5. distribution 없는 xlsx 업로드 시 탭에 "이미지 없음" 표시 + 업로드
   자체는 성공하는지(회귀) 확인.
